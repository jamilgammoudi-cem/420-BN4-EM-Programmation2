Instanciation de classe avec new

  1. Instanciation : création d'un objet avec new
  2. Accéder aux membres
    • propriétés : des valeurs appartenant à l'objet
    • méthodes : des actions ou traitements que l'objet peut effectuer
    • Constantes : lorsqu'une classe définit des constantes publiques, l'accès à ces constantes en dehors de la classe se fera toujours en utilisant le nom de la classe et non le nom de l'objet.
      exemple :  int valeur = MaClasse.NomConstante;
    • Méthodes statiques : lorsque la classe offre une ou plusieurs méthodes statiques, elles seront accessibles en utilisant le nom de la classe et non le nom de l'objet.
  3. Type valeur et type référence
      • Les variables qui stockent des valeurs en mémoire sont dites de type valeur (comme les types  int, char, decimal, bool,...)
      • Les variables qui stockent les adresses des objets en mémoire sont dites de type référence (comme les string, tableaux et classes)
      • Référence null : Si l'objet ne fait référence à aucun objet, sa valeur  null.  Un objet déclaré et non instancié a pour valeur null
  4. Variable locale et variable membre
    • Lorsqu'une variable est déclarée localement dans une méthode ou un bloc d'instructions, elle n'est visible est utilisable que dans la méthode ou le bloc où elle est déclarée. La variable est dite locale. Sa portée se limite à la méthode ou le bloc qui la déclare. Dès que l'exécution de la méthode ou du bloc est terminée, la variable n'existe plus.
    • Dans c# on peut déclarer des variables dont la portée est plus large (la classe). On dit que c'est une variable membre. Sa portée est toute la classe. Elle est utilisable dans toutes les méthodes de la classe. Pour les distinguer des variables locales, on adoptera le standard de les préfixer de m_ (m_prix par exemple). 
  5. Utilisation du débogueur pour visualiser les membres d'un objet
    • Lors de l'exécution, en mettant un point d'arrêt et en mettant le curseur sur l'objet, vous pouvez visualiser les valeurs des différentes propriétés de l'objet
    • Vous pouvez aussi voir la structure de la classe en faisant un clic droit sur le type de l'objet c'est à dire le nom de la classe et choisir "Atteindre la définition" dans le menu.

Détails de la classe DisqueCompact
  • Il y a 2 surcharges du constructeur de la classe DisqueCompact
    • Le premier sans paramètre permet d'obtenir un disque avec un nombre de pistes et des durées de pistes qui sont aléatoires
    • Le deuxième permet de spécifier le type disque par exemple : DisqueCompact objDisque = new DisqueCompact(SorteDeDisque.Test1);
      • L'énumération SorteDeDisque offre 4 choix possibles, le premier est Normal pour obtenir un disque aléatoire les 3 autres sont fixes soit :
        • SorteDeDisque.Test1 est un disque de 2 pistes, la première de 59 secondes et la deuxième de 5:01
        • SorteDeDisque.Test2 est un disque de 8 pistes chacune d'une durée de 5 minutes
        • SorteDeDisque.Test3 est un disque de 12 pistes alternant séquentiellement entre 3:59 et 4:01
  • La propriété Durée fournit la durée total en secondes de toutes les pistes du disque
  • La propriété Genre fournit le genre de musique du disque, pour l'obtenir en chaîne faire Genre.ToString()
  • La propriété Pistes fournit un tableau contenant la durée de chacune des pistes
    • Pistes.Length fournit le nombre de pistes que contient le disque
    • Pour obtenir la durée d'une piste en particulier, il faut indexer la propriété Pistes
      • Pistes[0] c'est la durée en secondes de la première piste
      • Pistes[1] c'est la durée en secondes de la deuxième piste
  • Les méthodes FormatMMSS et FormatHHMMSS permettent respectivement de formater une durée en secondes 
    • par exemple : string duréeFormatée = DisqueCompact.FormatMMSS(duréeEnSecondesÀFormater);

  4. Pourquoi parcourir une liste à partir de la fin pour retirer des éléments fonctionne correctement ?
    • Si on retire un élément d'une liste cela va raccourcir la liste et déplacer les éléments suivants vers l'élément retiré. Le problème c'est que le nouvel élément à la position index pourrait lui aussi devoir être retiré, mais l'index a été augmenté. Pour vous convaincre de ce problème, simplement recopier les deux exemples ci-dessous et les exécuter, ensuite observer le contenu de la collection après l'exécution de la boucle. Dans le premier cas seulement le 26 sera retiré, comme si on avait oublié le 14.
    • Prenons une liste générique de nombres entiers et essayons de retirer tous les nombres pairs en débutant au début 
      List<int> m_colNombres = new List<int>() { 26,14,23 };
 
// Version #1 qui ne fonctionne pas correctement
for (int index = 0; index < m_colNombres.Count; index++)
{
    if (m_colNombres[index] % 2 == 0)
        m_colNombres.RemoveAt(index);
}
    • Prenons une liste générique de nombres entiers et essayons de retirer tous les nombres pairs en débutant par la fin 
      List<int> m_colNombres = new List<int>() { 26,14,23 };
 
// Version #2 qui fonctionne correctement
for (int index = m_colNombres.Count -1; index >= 0; index--)
{
    if (m_colNombres[index] % 2 == 0)
        m_colNombres.RemoveAt(index);
}

Cliquez pour télécharger la solution

  1. Encapsulation : Protéger les données membres pour assurer l'intégrité de l'objet
    En programmation, l'encapsulation désigne le principe de regrouper les données (variables membres privées) avec un ensemble de routines (propriétés et/ou méthodes) permettant de les consulter ou de les manipuler. Dans ce cours nous allons protéger les données des objets afin d’assurer leur intégrité. Par exemple, la classe Thermostat qui modélise un objet de la vie réelle, ne devrait pas permettre d’augmenter la température à 999 ou encore de la fixer à -999 degrés Celsius.
  2. get : accès en lecture d'une propriété
    • Propriété en lecture seule 
      private int m_heures; // champ 

public int Heures // propriété 
{
    get // accès en lecture
    {
        return m_heures;
    }
}
  3. set : accès en écriture d'une propriété, utilisation du mot clé value
    • Propriété en écriture seule 
      private int m_heures; // champ 

public int Heures // propriété 
{
    set // accès en écriture
    {
        m_heures = value;
    }
}
    • Propriété en lecture et écriture 
      private int m_heures; // champ 

public int Heures // propriété 
{
    get // accès en lecture
    {
        return m_heures;
    }
    set // accès en écriture
    {
        m_heures = value;
    }
}
  4. Propriété calculée : propriété qui fournit une valeur à partir de champs existants
    • Une méthode provenant de la classe Montre sans paramètre qui produit un résultat 
      public string ObtenirTempsCourant()
{
    return m_heures + ":" +
            m_minutes.ToString("00") + ":" +
            m_secondes.ToString("00");
}
    • Une propriété calculée qui remplace la méthode ObtenirTempsCourant() 
      public string TempsCourant
{
    get
    {
        return m_heures + ":" +
                m_minutes.ToString("00") + ":" +
                m_secondes.ToString("00");
    }
}
    • Utilisation de la méthode versus utilisation de la propriété calculée 
      m_objMontre = new Montre(9, 05, 42);
 
// version utilisant la méthode sans paramètre
txtAffichageMontre.Text = m_objMontre.ObtenirTempsCourant();
 
// version utilisant la propriété calculée
txtAffichageMontre.Text = m_objMontre.TempsCourant;
  5. Propriété automatique : elle élimine la déclaration de la variable membre privée, mais elle ne permet pas de valider la valeur assignée avec l'accesseur en écriture.
    • Version habituelle incluant la déclaration d'une variable membre privée 
      private double m_temperature;
 
public double Temperature
{
    get { return m_temperature; }
}
    • Version utilisant la propriété automatique 
      // Propriété automatique en lecture publique et en écriture privé
public double Temperature { get; private set; }
  6. throw : lancer une exception lorsque l'état de l'objet est incohérent
    • Exemple #1 : un constructeur qui doit respecter la cohérence de l'objet 
      public Thermostat(double pTemperature)
{
    if (pTemperature < MIN_TEMPERATURE || pTemperature > MAX_TEMPERATURE)
        throw new ArgumentOutOfRangeException();
 
    m_temperature = pTemperature;
}
    • Exemple #2 : une propriété qui doit respecter la cohérence de l'objet 
      public int Heures
{
    get // accès en lecture
    {
        return m_heures;
    }
    set // accès en écriture
    {
        if (value < 0 || value > 23)
        {
            throw new ArgumentOutOfRangeException();
        }
        m_heures = value;
    }
}
